# Reader — Book Summary & Lecture Notes Vault

An Obsidian vault for comprehensive book summaries and lecture notes.

## Project Structure

```
inbox/                        — Drop raw files here (PDF, EPUB, MOBI for books; TXT for lecture transcripts)
analysis/                     — Intermediate analysis files (generated by /analyse or /lecture-analyse)
summaries/                    — Obsidian vault root — ALL readable content lives here
  <Category>/                 — Book summaries by category (e.g. Power & Influence/)
  lectures/                   — Lecture summaries organised by series
    Civilization/             — Prof. Jiang Xueqin's 60-lecture civilization history series
    Geo-Strategy/             — Prof. Jiang Xueqin's 12-lecture geopolitics series
    Secret History/           — Prof. Jiang Xueqin's 28-lecture series on power, evil, and collapse
```

## Workflows

### Books (Two-step process)

1. **`/analyse <filename>`** — Reads the book from `inbox/`, produces a structured analysis in `analysis/`
2. **`/summarize <reference>`** — Reads the analysis + original source, produces a comprehensive summary in `summaries/`

### Lectures (Two-step process)

1. **`/lecture-analyse <search term>`** — Reads a transcript from `inbox/`, cleans it, produces a structured analysis in `analysis/`
2. **`/lecture-summarize <reference>`** — Reads the analysis, produces a diagram-rich summary in `summaries/Lectures/<Series>/`
3. **`/lecture-process <search term>`** — Runs the full pipeline (analyse → summarize) for one or all lectures

## File Naming Conventions

- **Book summaries:** `Title - Author.md` — e.g. `Thinking Fast and Slow - Daniel Kahneman.md`
- **Book analysis:** `author-lastname_short-title.md` — e.g. `kahneman_thinking-fast-slow.md`
- **Lecture summaries:** `NN - Title.md` — e.g. `01 - Explaining Humanity's Transition to Agriculture.md`
- **Lecture analysis:** `jiang_series-NN.md` — e.g. `jiang_civilization-01.md`

Main title only (no subtitle) for books. Lecture numbers are zero-padded.

## Summary Standards

- Comprehensive, not condensed — 500-1200+ lines depending on book type
- Three reading depths: 30-second scan, 5-minute review, full read
- Story-driven with real names, dates, specifics
- Plain English, no academic formatting
- Copyright-compliant (quotes under 15 words, stories paraphrased)
- Cross-referenced using `[[wikilinks]]`

## Formatting Rules

These rules apply to all summaries (existing and new):

- **Colored text is always bold** — use `<b style="color: #hex">text</b>`, never `<span>`. The three standard colors are: `#2980b9` (blue, primary insight), `#e74c3c` (red, problem/danger), `#27ae60` (green, solution/antidote)
- **Tables** use standard markdown pipe syntax with header row and separator
- **Callouts** use Obsidian `> [!type] Title` syntax. Supported types: example, tip, warning, danger, success, info, quote, note
- **Mermaid diagrams** use fenced code blocks with `mermaid` language tag
- **Wikilinks** use `[[Title - Author]]` or `[[Title - Author|Display Text]]` format with no path prefix

## Rules

- Always read `summaries/_index.md` before creating a new book summary to check for existing entries
- Always update `summaries/_index.md` after completing a book analysis or summary
- Always update `summaries/Lectures/<Series>/_overview.md` after completing a lecture summary
- Never reproduce large sections of copyrighted text
- Wikilinks use `[[filename|Display Title]]` format with no path prefix
- Lecture summaries require minimum 4 Mermaid diagrams (target 6-8) — see `.claude/rules/lecture-summary-standards.md`
- Lecture transcripts in `inbox/` are identified by `Civilization #N;`, `Geo-Strategy #N;`, or `Secret History #N;` filename patterns
