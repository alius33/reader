# Reader — Book Summary Vault

An Obsidian vault for comprehensive book summaries.

## Project Structure

```
inbox/          — Drop raw book files here (PDF, EPUB, MOBI)
analysis/       — Intermediate analysis files (generated by /analyse)
summaries/      — Finished human-readable summaries (generated by /summarize)
```

## Workflow

Two-step process:

1. **`/analyse <filename>`** — Reads the book from `inbox/`, produces a structured analysis in `analysis/`
2. **`/summarize <reference>`** — Reads the analysis + original source, produces a comprehensive summary in `summaries/`

## File Naming Convention

`Title - Author.md` — e.g. `Thinking Fast and Slow - Daniel Kahneman.md`

Main title only (no subtitle). Used consistently across `analysis/` and `summaries/`.

## Summary Standards

- Comprehensive, not condensed — 500-1200+ lines depending on book type
- Three reading depths: 30-second scan, 5-minute review, full read
- Story-driven with real names, dates, specifics
- Plain English, no academic formatting
- Copyright-compliant (quotes under 15 words, stories paraphrased)
- Cross-referenced using `[[wikilinks]]`

## Rules

- Always read `summaries/_index.md` before creating a new summary to check for existing entries
- Always update `summaries/_index.md` after completing an analysis or summary
- Never reproduce large sections of copyrighted text
- The Obsidian vault root is `summaries/` — wikilinks inside summaries use `[[filename|Display Title]]` with no path prefix
