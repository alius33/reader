# Audio Script Standards — ElevenLabs Production

Shared rules for all audio scripts generated by `/book-audio-script` and `/lecture-audio-script`.

---

## ElevenLabs Formatting Rules

### What survives in the output file

- `[Narrator]:` `[Author]:` `[Prof. Jiang]:` `[Student]:` speaker labels
- `<break time="Xs" />` pause tags
- `---` section dividers (become ElevenLabs chapter boundaries)
- Standard punctuation: `.` `,` `?` `!` `...` `—`
- Paragraph breaks (blank lines)
- YAML frontmatter
- The production notes comment block `<!-- ... -->`

### What must NOT survive

Run this mental checklist before saving. If any of these patterns remain, fix them:

| Pattern | Example | Action |
|---------|---------|--------|
| Headings | `## Strategy 1` | Convert to verbal transition |
| Bold/italic | `**term**`, `*text*` | Read as plain text |
| Colour HTML | `<b style="color:...">` | Drop tags, keep text |
| Bullet points | `- item` | Convert to prose |
| Numbered lists | `1. step` | Convert to prose or verbal list |
| Blockquotes | `> text` | Convert to inline narrative |
| Callout boxes | `> [!example]` | Convert to "Here's the story..." |
| Mermaid diagrams | ` ```mermaid ` | Books: skip. Lectures: convert to verbal. |
| Wikilinks | `[[Title]]` | Books: "as we covered in [Title]". Lectures: absorbed into connections. |
| Table pipes | `\| col \|` | Convert to spoken comparisons |
| HTML tags | `<b>`, `<span>`, `<br>` | Drop entirely |
| Frontmatter lines | `---\ndate:...` | Keep YAML block; strip from body |
| Code fences | ` ``` ` | Drop entirely |

## Break Tag Reference

| Tag | Duration | Use when |
|-----|----------|----------|
| `<break time="0.4s" />` | Short | Between sentences in a dense passage |
| `<break time="0.8s" />` | Medium | Between paragraphs within a section |
| `<break time="1.2s" />` | Long | Between major sections within a chapter |
| `<break time="2.0s" />` | Chapter break | Between `---` act dividers (after the divider) |

ElevenLabs Projects respects paragraph breaks as natural pauses — explicit break tags are for larger structural pauses. Do not over-annotate.

## Speaker Label Reference

| Label | Used in | Role |
|-------|---------|------|
| `[Narrator]:` | Books | All narration, explanation, stories, transitions |
| `[Author]:` | Books | Direct quotes from the book only (under 15 words) |
| `[Prof. Jiang]:` | Lectures | All lecture content (first person) |
| `[Student]:` | Lectures | Q&A student questions only (1-3 sentences max) |

Always put the label on its own prefix to each paragraph. Never mid-paragraph.

**Correct:**
```
[Narrator]: Greene opens this strategy with a deceptively simple claim.

[Author]: "Strategy is a way of seeing, not a set of recipes."

[Narrator]: That distinction is the axis around which the entire book turns.
```

**Wrong:**
```
[Narrator]: Greene opens this strategy with a deceptively simple claim. [Author]: "Strategy is a way of seeing." [Narrator]: That distinction...
```

## Adaptation Rules: Element by Element

### All content types

| Source element | Audio treatment |
|---------------|----------------|
| `## Chapter heading` | Verbal transition: "Let's turn now to..." — never read as a title |
| `### Sub-heading` | Absorbed into prose as a transition sentence |
| `**Bold term**` | First mention: emphasised through word choice ("what Greene calls..."); after that: plain |
| `*Italic section opener*` | Drop the italic; absorb the content as an opening sentence |
| `<b style="color: #e74c3c">text</b>` | Red = warning/danger: read as plain text, but precede with "Greene warns that..." |
| `<b style="color: #27ae60">text</b>` | Green = key insight: read as plain text, perhaps precede with "The core principle here is..." |
| `<b style="color: #2980b9">text</b>` | Blue = named concept: read as plain text, introduce the name naturally |
| `- Bullet point` | Convert to prose sentence or natural list |
| Nested bullets | Convert to flowing paragraph with connecting phrases |
| `> [!example] Story Title` | "Here's the story of [person/event]..." then narrative prose; drop the title |
| `> [!tip] Insight` | Absorbed as a key point in narration; never announced as "the tip" |
| `> [!abstract] Method` | "Here's how [method] works..." then step-by-step narration |
| `> [!warning] ...` | "A word of warning here..." then the warning in prose |
| `> [!note] ...` | Absorbed as a parenthetical or aside in narration |
| `\| Table \|` | Spoken as comparisons: "The first approach... the second approach..." |
| `[[Wikilink]]` | Books: "as covered in [display title]". Lectures: absorbed into connections. |
| `[[Title\|Display]]` | Use the display text only |
| Frontmatter | Ignored in the prose body |
| `---` horizontal rule | `<break time="1.2s" />` then blank line |

### Books only

| Source element | Audio treatment |
|---------------|----------------|
| Mermaid diagrams | **SKIP entirely** — visual only |
| About the Author section | Absorbed into Intro (narrator biog paragraph) |
| 30-second blockquote at top | Absorbed into Intro or Big Idea |
| Key Concepts at a Glance table | Each concept becomes a verbal introduction: "The first concept Greene introduces is..." |
| Verdict section | Becomes the Verdict segment of the script |
| "**The lesson:**" in callouts | Weave into the narrative close of the story — never read as a label |

### Lectures only

| Source element | Audio treatment |
|---------------|----------------|
| Mermaid diagrams | **CONVERT** — never skip. See diagram conversion rules in lecture skill. |
| Theory evaluation tables | Become spoken theory-by-theory evaluation with verdict |
| Connections section | Intro absorbs "previous lecture" reference; Outro absorbs "next lecture" preview |
| Q&A section | Two-voice dialogue (Prof. Jiang + Student) |
| Series overview references | Absorbed into Outro |

## Prose Quality Standards

### Sentence construction

- Vary sentence length: mix short punchy sentences with longer explanatory ones
- Avoid starting consecutive sentences with "Greene" or "He" — mix subject positions
- Use the second person "you" regularly: "What this means for you is..." / "The question you have to ask yourself..."
- Rhetorical questions: maximum 2-3 per major section — then answer them immediately
- Em-dashes for spoken rhythm: "The strategy — and this is the part people miss — depends entirely on..."

### Transitions between sections

Do not use heading-style transitions. Use these natural spoken forms:

- "Let's move now to [topic]..."
- "From here, Greene turns to a related problem..."
- "This brings us to what I consider the book's most important idea..."
- "Chapter [N] addresses something that the previous argument left unresolved..."
- "Now, with that foundation in place, Greene introduces..."
- "There's a counterintuitive insight hiding in [topic]..."

### List handling

Never read bullets as bullets. Convert:

| From | To |
|------|----|
| "First: X. Second: Y. Third: Z." | "The approach has three components. X, which handles... Then Y, which creates... And finally Z, where the whole structure converges." |
| "- Napoleon\n- Musashi\n- Boyd" | "Three strategists in particular — Napoleon, Musashi, and John Boyd — all arrived at the same conclusion independently." |

## Runtime Estimation Guide

| Summary lines | Approx audio prose words | Target runtime |
|--------------|--------------------------|---------------|
| Under 500 | ~3,500–5,000 | 30–45 min |
| 500–900 | ~5,000–8,000 | 45–60 min |
| 900–1,400 | ~8,000–12,000 | 60–80 min |
| Taxonomy: 33 entries | ~8,000–10,000 core + frameworks | ~90–120 min |
| Taxonomy: 48 entries | ~10,000–14,000 | ~110–150 min |

Professional narration benchmark: ~150 words/minute at natural pace.

## Standardised Intro & Outro Templates

### Book intro
```
[Narrator]: Welcome. Today we're exploring [Title] by [Author], published in [Year]. [1-2 sentence hook — specific, not generic]. By the end of this, you'll understand [the book's central promise or transformation].

[Narrator]: [Author] is [2-3 sentences: relevant credentials, background, why qualified, one memorable detail].
```

### Book outro
```
[Narrator]: That's [Title] by [Author]. [One final thought — the book's lasting contribution in one sentence, or the single question it leaves you with].

<break time="1.0s" />

[Narrator]: Thank you for listening.
```

### Lecture intro
```
[Prof. Jiang]: Welcome back. This is Lecture [N] of [Series Name]. Today's question — [the central question, stated as Prof. Jiang would state it: direct, slightly provocative, framed as a genuine mystery].

[Prof. Jiang]: Last time, we established [1-2 sentences connecting to previous lecture]. Today's question builds directly on that.

<break time="1.0s" />
```

### Lecture outro
```
[Prof. Jiang]: That's Lecture [N]: [Title]. Next time, we'll be asking [next lecture's central question]. I'll see you then.

<break time="1.5s" />
```
